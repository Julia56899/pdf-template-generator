Тема:
Разработка сервиса генерации PDF документов с использованием Python и Javascript


Описание проекта: в рамках данной работы был разработан минималистичный веб сервис для создания PDF документов на основе HTML шаблонов с поддержкой подстановки пользовательских значений.

Пользователь может:
- создавать HTML шаблоны через визуальный редактор
- сохранять шаблоны в базе данных
- редактировать и удалять шаблоны
- использовать маркеры вида {value}
- выполнять подстановку данных
- просматривать HTML результат
- генерировать PDF документ
- просматривать PDF прямо в интерфейсе приложения


Функциональные возможности:

1. Работа с HTML шаблонами.
   Пользователь может создавать и редактировать HTML шаблоны через визуальный WYSIWYG редактор Quill.js.

2. Подстановка шаблонных значений.
   В шаблонах используются маркеры вида:
   "пример"
{name}
{group}
{date}

Во время генерации документа сервис автоматически заменяет маркеры на пользовательские значения.

3. Генерация PDF документов.
   После подстановки значений итоговый HTML преобразуется в PDF документ при помощи библиотеки xhtml2pdf.

4. HTML preview.
   Перед генерацией PDF пользователь может посмотреть итоговыи HTML документ с уже подставленными значениями без скачивания PDF файла.

5. Хранение шаблонов.
   Все шаблоны сохраняются во встроенной базе данных SQLite.

6. CRUD операции.
   Система поддерживает:
- Create — создание шаблона
- Read — просмотр шаблонов
- Update — редактирование шаблона
- Delete — удаление шаблона


Структура проекта:

project/

├── app.py  
Файл backend части приложения. Содержит Flask сервер, API endpoints, работу с SQLite, подстановку шаблонов и генерацию PDF документов.

SQLite база данных, в которой хранятся HTML шаблоны пользователя.

Шрифт для корректного отображения кириллицы в PDF документах.

├── templates/  
Папка с HTML шаблонами Flask приложения.

│   └── index.html  
Основной пользовательский интерфейс приложения. Содержит HTML, CSS и Javascript код frontend части.

└── README.md  
Документация проекта с описанием функциональности, API и инструкции по запуску.


Описание API endpoints:

1. GET /
Возвращает HTML интерфейс приложения.

2. GET /templates
Возвращает список всех шаблонов.

3. POST /templates
Создает новый шаблон.

4. GET /templates/{id}
Возвращает шаблон по ID.

5. PUT /templates/{id}
Обновляет существующий шаблон.

6. DELETE /templates/{id}
Удаляет шаблон.

7. POST /render/{id}
Выполняет подстановку значений и генерирует PDF документ.

8. GET /preview/{id}
Возвращает HTML preview с подстановкой значении.

Проверка API endpoints выполнялась через Postman и через web интерфейс приложения.


Особенности реализации:

1. WYSIWYG редактор.
   Для визуального редактирования HTML использована библиотека Quill.js. Редактор позволяет работать с HTML контентом без ручного написания тегов.

2. Работа с PDF.
   PDF документы генерируются через библиотеку xhtml2pdf (Pisa), преобразующую HTML документ в PDF формат.

3. Поддержка русских символов.
   Для корректного отображения кириллицы используется шрифт DejaVuSans.ttf, так как библиотека xhtml2pdf не всегда корректно отображает стандартные шрифты.

   4. Подстановка значений через template strings.
      Подстановка значений реализована через механизм Template в Python. Перед генерацией PDF сервис автоматически заменяет шаблонные маркеры на пользовательские значения.


Запуск проекта:

1. Установить зависимости

pip install flask xhtml2pdf

2. Запустить приложение

python app.py

3. Открыть в браузере ресурс 

http://127.0.0.1:5000


Пример работы системы:

1. Пользователь создает HTML шаблон.
2. В шаблон добавляются маркеры вида {name}.
3. Шаблон сохраняется в SQLite.
4. Пользователь вводит значения для маркеров.
5. Сервис выполняет подстановку данных.
6. Генерируется PDF документ.
7. PDF отображается в preview окне и может быть скачан.


Использованные технологии:

 - Python 3.14 (Основной язык разработки backend части приложения)

 - Flask (Мини веб фреймворк для создания web сервера и API endpoints)

 - SQLite (Встроенная база данных для хранения HTML шаблонов без необходимости отдельного сервера БД)

 - HTML (Язык разметки пользовательского интерфейса)

 - CSS (Язык стилей для оформления интерфейса приложения)

 - Javascript (Используется для работы кнопок, запросов к backend и обновления интерфейса без перезагрузки страницы)

 - Quill.js (WYSIWYG редактор для визуального редактирования HTML контента)

 - xhtml2pdf (Pisa) (Библиотека для генерации PDF документов из HTML)

 - string.Template (Механизм template strings в Python для подстановки пользовательских значений в HTML шаблон)

 - Postman (Инструмент для тестирования API endpoints)
